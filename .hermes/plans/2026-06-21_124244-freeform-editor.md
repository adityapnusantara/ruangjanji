# Plan: Ubah Editor dari Sequential Steps ke Free-Form Panel

## Goal
Ganti flow editor yang sekarang **wajib step-by-step** (Cover → Mempelai → Acara → Galeri → Story → Tambahan → Review) jadi **free-form** — user bebas klik bagian mana aja yang mau diisi.

## Current Context
Editor sekarang di `InvitationEditor.tsx` — single page dengan state `step` (index 0-6), navigasi prev/next, tiap step render satu komponen. Data tersimpan di `InvitationFormData` state.

## Proposed Approach

### Layout baru

```
┌─────────────────────────────────────────────────┐
│  Sidebar (menu)               │ Panel (konten)  │
│                               │                 │
│  ✓ Cover & Judul              │ [form untuk     │
│  ✓ Mempelai                   │  section yang   │
│  ✓ Acara                      │  dipilih]       │
│  ✓ Galeri                     │                 │
│  ✓ Story                      │                 │
│  ✓ Tambahan                   │                 │
│  ✓ Review & Publish          │                 │
│                               │                 │
│  [Simpan Draft] [Publish]     │                 │
└─────────────────────────────────────────────────┘
```

- Sidebar kiri (lebar ~260px) — daftar section dengan status
- Panel kanan — form untuk section yang diklik
- Sidebar selalu kelihatan, user bisa loncat kapan aja
- Tombol Simpan & Publish di sidebar, bukan di dalam panel

### Files to change

| File | Change |
|---|---|
| `src/app/dashboard/invitations/InvitationEditor.tsx` | **Rewrite** — ganti step machine jadi sidebar + panel |
| `src/app/dashboard/invitations/new/page.tsx` | Mungkin minor, cek layout |
| `src/app/dashboard/invitations/[id]/edit/page.tsx` | Mungkin minor, cek layout |

### Komponen yang tetap

- `CoverStep`, `CoupleStep`, `EventsStep`, `GalleryStep`, `StoryStep`, `ExtraStep`, `ReviewStep` — konten formnya TIDAK berubah, cuma cara navigasinya aja
- `createInvitation`, `updateInvitation`, `publishInvitation` — server actions tetap

## Implementation Steps

1. **Restruktur InvitationEditor.tsx:**
   - Hapus `step` state (number), ganti `activeSection` state (string: 'cover' | 'couple' | 'events' | 'gallery' | 'story' | 'extra' | 'review')
   - Hapus step indicator + prev/next buttons
   - Tambah sidebar component dengan daftar section, klik → set activeSection
   - Sidebar kasih ikon checklist kalo section udah diisi
   - Panel kanan render komponen sesuai `activeSection`
   - Tombol Simpan Draft & Publish pindah ke fixed bottom sidebar

2. **Status tracking per section:**
   - Fungsi `isSectionFilled(section)` — cek apakah data section minimal terisi (nama/url ada value)
   - Sidebar show checklist hijau / abu-abu

3. **Layout CSS:**
   - Grid 2 kolom: sidebar (260px) + panel (1fr)
   - Sidebar sticky, scroll panel independen

## Validation
- `npm run build` — 0 error
- Bikin undangan baru → bisa klik section bebas
- Edit undangan → data existing muncul
- Simpan → semua section tersimpan

## Risks
- Layout change butuh tes visual — pastikan mobile friendly (sidebar collapse?)
- State masih di `InvitationFormData` yang sama — aman
- Event listener untuk section check perlu di-refresh tiap kali user ngetik

## Open Questions
